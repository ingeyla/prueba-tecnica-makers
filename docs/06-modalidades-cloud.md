# Modalidades Cloud

## Single-cloud

- Scope: un track (`aws` o `gcp` o `azure`).
- Objetivo: profundidad técnica en un proveedor.

## Dual-cloud

- Scope: dos tracks.
- Objetivo: comparar decisiones, abstraer patrones y evitar lock-in accidental.

## Multi-cloud

- Scope: tres tracks.
- Objetivo: estandarizar baseline transversal con variaciones mínimas por proveedor.
